Project Phases and Responsibilities:
  • Phase 1: Requirement Analysis and Feasibility Study
    • Conduct a detailed requirement analysis to understand the specific needs and constraints of the system.
    • Perform a feasibility study to evaluate the technical and operational viability of the project.
    • Responsible: Muhammad Sannan
  • Phase 2: System Design
    • Design the system architecture, including hardware and software components.
    • Define the data pipeline for collecting, preprocessing, and analyzing air pollution data.
    • Responsible: Atta Ur Rehman
  • Phase 3: Data Collection and Preprocessing
    • Collect historical and real-time air pollution data from various sources such as sensors, weather stations, and public databases.
    • Preprocess the data to remove noise, handle missing values, and normalize the data.
    • Responsible: Muhammad Rizwan
  • Phase 4: Model Development
    • Implement a multivariate multistep time series model using the random forest algorithm.
    • Train the model on the collected data to predict future air pollution levels.
    • Responsible: Muhammad Rizwan
  • Phase 5: System Integration
    • Integrate the trained model into the overall system architecture.
    • Develop a user-friendly interface for environmental protection departments to interact with the system and view predictions.
    • Responsible: Atta Ur Rehman + Rimsha
  • Phase 6: Testing and Evaluation
    • Conduct extensive testing to evaluate the accuracy and robustness of the air pollution prediction model.
    • Perform validation using real-world data to ensure effectiveness in practical scenarios.
    • Responsible: Muhammad Rizwan
  • Phase 7: Deployment and Maintenance
    • Deploy the system in the target environment.
    • Establish a maintenance plan to ensure long-term reliability and performance.
    • Responsible: Mamoon Zaheer & Maryam Wahab
